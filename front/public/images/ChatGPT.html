1. Contexto en React

El Context API de React permite compartir estados o funciones entre múltiples componentes sin necesidad de pasarlos manualmente como props. Esto es útil cuando varias partes de la aplicación necesitan acceder a la misma información, como el tema, la autenticación o los datos del usuario.

Ejemplo de creación y uso de un contexto:

jsx
import React, { createContext, useState, useContext } from "react";

// 1. Crear el contexto
const UserContext = createContext();

// 2. Proveedor del contexto
const UserProvider = ({ children }) => {
  const [user, setUser] = useState({ name: "Juan", loggedIn: true });

  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  );
};

// 3. Hook para usar el contexto en cualquier componente
const useUser = () => {
  return useContext(UserContext);
};

// 4. Componente que consume el contexto
const UserProfile = () => {
  const { user } = useUser();

  return <h2>Hola, {user.name}!</h2>;
};

// 5. Integrar el proveedor en la app
const App = () => {
  return (
    <UserProvider>
      <UserProfile />
    </UserProvider>
  );
};

export default App;

2. Custom Hook

Un Custom Hook es una función de JavaScript que usa hooks internos de React para encapsular lógica reutilizable.

Ejemplo: Un custom hook para manejar el estado local de un contador.

jsx
import { useState } from "react";

// Hook personalizado
const useCounter = (initialValue = 0) => {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);
  const reset = () => setCount(initialValue);

  return { count, increment, decrement, reset };
};

// Ejemplo de uso en un componente
const Counter = () => {
  const { count, increment, decrement, reset } = useCounter(10);

  return (
    <div>
      <h2>Contador: {count}</h2>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
};

export default Counter;

Combinando un Contexto con un Custom Hook

Ahora, unimos ambas ideas creando un contexto para manejar el estado global de un contador y un custom hook para consumirlo.

jsx
import { createContext, useContext, useEffect, useState } from "react";

// 1. Crear el contexto
const CustomerContext = createContext();

// 2. Proveedor del contexto
export const CustomerProvider = ({ children }) => {
  const username = localStorage.getItem("username") || ""; // Asegura que no sea null
  const [favorites, setFavorites] = useState([]);
  const [error, setError] = useState("");

  useEffect(() => {
    if (!username) {
      console.warn("No hay username, no se hace la petición");
      return;
    }

    const getUserFavorites = async () => {
      try {
        const url = `http://localhost:3001/favorites?username=${encodeURIComponent(username)}`;
        console.log("URL de la petición:", url);

        const response = await fetch(url);
        if (!response.ok) {
          throw new Error("No se pudieron obtener los favoritos");
        }

        const result = await response.json();
        console.log("Resultado del backend en el front:", result);

        if (!result.favorites) {
          console.warn("No hay favoritos en la respuesta.");
          setFavorites([]);
          return;
        }

        setFavorites(result.favorites);
      } catch (error) {
        console.error("Error al obtener los favoritos:", error);
        setError(error.message);
      }
    };

    getUserFavorites();
  }, [username]);

  return (
    <CustomerContext.Provider value={{ username, favorites, error }}>
      {children}
    </CustomerContext.Provider>
  );
};

// 3. Custom Hook para usar el contexto
export const useCustomerArea = () => {
  return useContext(CustomerContext);
};

  const [favorites, setFavorites] = useState([]);
  const [error, setError] = useState("");

  useEffect(() => {
    if (!username) {
      console.warn("No hay username, no se hace la petición");
      return;
    }

    const getUserFavorites = async () => {
      try {
        const url = `http://localhost:3001/favorites?username=${encodeURIComponent(username)}`;
        console.log("URL de la petición:", url);

        const response = await fetch(url);
        if (!response.ok) {
          throw new Error("No se pudieron obtener los favoritos");
        }

        const result = await response.json();
        console.log("Resultado del backend en el front:", result);

        if (!result.favorites) {
          console.warn("No hay favoritos en la respuesta.");
          setFavorites([]);
          return;
        }

        setFavorites(result.favorites);
      } catch (error) {
        console.error("Error al obtener los favoritos:", error);
        setError(error.message);
      }
    };

    getUserFavorites();
  }, [username]);

  return (
    <CustomerContext.Provider value={{ username, favorites, error }}>
      {children}
    </CustomerContext.Provider>
  );
};

// 3. Custom Hook para consumir el contexto
export const useCustomerArea = () => {
  return useContext(CustomerContext);

❌ ¿Por qué useCourses() no es un contexto?

Actualmente, useCourses() es un custom hook que:
✅ Usa useState y useEffect para manejar el estado de los cursos.
✅ Hace una petición fetch() cada vez que un componente lo llama.
✅ Devuelve el estado courses, pero no comparte los datos globalmente.

🔴 Problema:
Si useCourses() se usa en varios componentes, cada uno hará su propia llamada a la API, lo que puede ser ineficiente.

✅ Solución: Convertir useCourses() en un Contexto

📌 1️⃣ Crear CoursesContext.js

jsx
import { createContext, useContext, useEffect, useState } from "react";

// 1. Crear el contexto
const CoursesContext = createContext();

// 2. Crear el proveedor del contexto
export const CoursesProvider = ({ children }) => {
  const [courses, setCourses] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchCourses = async () => {
      try {
        const response = await fetch("http://localhost:3001/creanunate/courses/all-courses");
        if (!response.ok) {
          throw new Error(`Error en la respuesta del servidor: ${response.status}`);
        }
        const data = await response.json();
        console.log("Cursos obtenidos:", data);
        setCourses(data);
      } catch (err) {
        console.error("Error en la solicitud:", err);
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchCourses();
  }, []);

  return (
    <CoursesContext.Provider value={{ courses, loading, error }}>
      {children}
    </CoursesContext.Provider>
  );
};

// 3. Custom Hook para consumir el contexto
export const useCourses = () => {
  return useContext(CoursesContext);
};

✅ Centraliza la gestión del estado, manteniendo el código más organizado.
✅ Hace el código más reutilizable y modular.
✅ Mejor manejo de estados como loading y error, mejorando la experiencia de usuario.

💡 Ahora, cualquier componente puede acceder a los cursos sin hacer nuevas peticiones! 🚀

No necesariamente, pero es una convención común en React nombrar el proveedor del contexto con el sufijo "Provider".

📌 ¿Por qué "CoursesProvider"?

  • El contexto en sí mismo (CoursesContext) almacena los datos y permite que los componentes accedan a ellos.
  • El proveedor (CoursesProvider) es el que envuelve los componentes y proporciona los datos.

📌 ¿Puedo cambiar el nombre?

Sí, pero debes asegurarte de que los nombres sean coherentes en toda la aplicación. Por ejemplo:

🔹 Opción 1: Mantener "Provider" (Recomendado)
